Setting up Virtual environment in ubuntu (Probably the best way)#

Setting up virtual environment using virtual-environment-wrapper

Pre-requisites :

Python3
pip3

Install the ‘virtualenvwrapper’ using pip3: pip url : https://pypi.org/project/virtualenvwrapper/

~:$ pip3 install virtualenvwrapper

Then find the location of the python executables

~:$ which python3 
/usr/bin/python3

Then add the following lines to ‘.bashrc’ file These set the following locations

  • where the virtual environments should live,
  • the location of the script installed by virtualenvwrapper
export WORKON_HOME=$HOME/.virtualenvs
export VIRTUALENVWRAPPER_PYTHON=/usr/bin/python3
export VIRTUALENVWRAPPER_VIRTUALENV_ARGS=' -p /usr/bin/python3 '
source /usr/local/bin/virtualenvwrapper.sh

/usr/bin/python3 was selected form the output of the which command Select you python version to be used here, ie. if you want o use python 3.7 in th virtual environments select the executable path for python3.7

Then reload the startup file depending on the shell you are using by running the following command :

~:$ source ~/.bashrc

At this point you should see a bunch of scripts being run as shown below:

virtualenvwrapper.user_scripts creating /home/ubuntu/.virtualenvs/premkproject
virtualenvwrapper.user_scripts creating /home/ubuntu/.virtualenvs/postmkproject
...
virtualenvwrapper.user_scripts creating /home/ubuntu/.virtualenvs/preactivate
virtualenvwrapper.user_scripts creating /home/ubuntu/.virtualenvs/postactivate
virtualenvwrapper.user_scripts creating /home/ubuntu/.virtualenvs/get_env_details

Thats all , installation of virtualenvwrapper is compete , now we can use this for development

# syntax :   mkvirtualenv {virtual envornment name}
~:$ mkvirtualenv my_new_virtual_env

For listing Virtual environments :

~:$ workon
my_new_virtual_env

To activate any virtual environment , go to any directory and enter the following command:

~:$ workon my_new_virtual_env
(my_new_virtual_env) ~:$

To deactivate the virtual environment :

(my_new_virtual_env) ~:$ deactivate 
~:$

You can use this virtual environment in IDEs also , for that select python executable from the following path

# /home/{user}/.virtualenvs/{virtual env name}/bin/python3.6
/home/user/.virtualenvs/my_new_virtual_env/bin/python3.6
