Setting up Kubernets cluster#

A Three node cluster is going to set up and all machies are having Ubuntu 20.04 Installed and Updated

Prerequisites :#

VM with atleast 2GB of RAM and Ubuntu Ubuntu 20.04

Note : the following steps need to carried out in master and worker nodes

The following applications needs to be installed

  1. vim
  2. git
  3. curl
  4. wget
  5. apt-transport-https
  6. docker

Make sure sure system repositories and apps are updated

sudo apt update
sudo apt upgrade -y 
sudo apt install apt-transport-https vim git curl wget  gnupg2 software-properties-common ca-certificates -y

and restart your machine

Install Docker - as container run time

# Install Docker 

# Add docker repository
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"

#install docker 
sudo apt update
sudo apt install  containerd.io docker-ce docker-ce-cli -y

# Configure docker essentials 
sudo mkdir -p /etc/systemd/system/docker.service.d
sudo tee /etc/docker/daemon.json <<EOF
{
  "exec-opts": ["native.cgroupdriver=systemd"],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m"
  },
  "storage-driver": "overlay2"
}
EOF

# Add user to docker group
sudo groupadd docker
sudo usermod -aG docker $USER

# Start and enable Services
sudo systemctl daemon-reload 
sudo systemctl restart docker
sudo systemctl enable docker

Disable swap - why ?

sudo sed -i '/ swap / s/^\(.*\)$/#\1/g' /etc/fstab
sudo swapoff -a

Setting up master node#

Step 1#

Adding Kubernetes official repo to your local#

curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | sudo apt-key add -
echo "deb https://apt.kubernetes.io/ kubernetes-xenial main" | sudo tee /etc/apt/sources.list.d/kubernetes.list

Step 2#

Install Kubernetss and components#

sudo apt update
sudo apt install kubelet kubeadm kubectl -y

Confirm the installation :

kubectl version --client

Step 3#

Install/load essential kernal modules#

a) br_netfilter

sudo modprobe br_netfilter
sudo sh -c 'echo "br_netfilter" > /etc/modules-load.d/br_netfilter.conf'

This module is required to communication between Kubernetes pods across the cluster - Ref

for verifying this module is enabled tuse the command lsmod | grep br_netfilter , will get a sample value like

midhun@master:~$ lsmod | grep br_netfilter br_netfilter 28672 0 bridge 172032 1 br_netfilter

by seeing values like above , we can assure that module is installed

b) overlay

sudo modprobe overlay

This module is n intended for enabling overlay file system

c) br_netfilter

sudo tee /etc/sysctl.d/kubernetes.conf<<EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables = 1
net.ipv4.ip_forward = 1
EOF

Kubernetes requires that packets traversing a network bridge are processed for filtering and for port forwarding. will be used by kubeadm - Ref
